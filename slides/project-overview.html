CLAUDE.md

Route hub — Quick Commands, Agent table, Skills table, Workflow, Conventions. Stays under 80 lines. Never grows into docs.

Naming Conventions

Experiments: exp01a, exp01b, exp02a
Prompts: prompts/comp/_v01.md → _v02.md
Never overwrite — always increment.

.pipeline-state.json

Tracks current stage, current_exp, skill_updated_at. Hooks read it. Skills write it. Single source of truth.

Least-Privilege Design

7 Agents — Each Does One Thing

Agent Model Writes To Purpose
project-advisor Opus read-only Experiment history, codebase navigation
cc-advisor Sonnet read-only CC workflow and tooling best practices
domain-expert Opus read-only Deep domain reasoning, reads papers
slides-maker Sonnet slides/ Generate HTML slide presentations
exp-manager Sonnet Bash+retry Monitor, diagnose, retry experiments
viz-frontend Sonnet viewer/ Flask dashboards and matplotlib figures
template-presenter Sonnet read-only Template architecture docs and onboarding
4/7 read-only No agent ever touches exp/ or .claude/ directly — only Skills do.

Skill Definitions

4 Skills — Verbs, Not Nouns

/new-experiment
Scaffold exp/{id}/ with README, config.yaml, run.py, analyze.py, results/. Updates exp/summary.md and advances pipeline state to "experiment".
scaffold state→experiment
/analyze-experiment
Runs analyze.py → spawns domain-expert (Opus, read-only) → merges interpretation → spawns slides-maker (Sonnet). Full pipeline in one command.
Opus subagent Sonnet subagent
/update-project-skill
Opus subagent scans entire codebase → generates SKILL.md diff → user approves → writes + CHANGELOG entry + updates pipeline state.
Opus subagent diff-first
/present-template
Spawns template-presenter (Sonnet, read-only) to gather content, then slides-maker to generate HTML. Supports project-overview, onboarding, or custom topics.
2 subagents writes slides/
"Skills are verbs — Agents are nouns"
06 / 10
Automation Layer

5 Hooks — Remind, Don't Block

PreCompact pre-compact-remind Warns when project skill is >24h stale before context compacts
Stop stop-check-workflow Reads pipeline stage → reminds next action + checks CHANGELOG compliance
PostCommit post-commit-changelog After git commit — checks if prompts/ or .claude/skills/ changed without CHANGELOG
PreWrite brainstorm-remind New file creation without a recent spec in docs/specs/ → suggests /brainstorming
PreBash worktree-suggest Destructive git ops (reset --hard, clean -fd) → suggests git worktree instead
"All hooks suggest — never hard-block"
07 / 10
Workflow

Experiment Lifecycle

Phase 1
Scaffold
  • Run /new-experiment
  • Name: exp{NN}{x} format
  • Creates: README, config.yaml, run.py, analyze.py
  • Updates exp/summary.md + pipeline state
Phase 2
Execute & Monitor
  • Run python scripts/launch_exp.py
  • Loop every 5 min via exp-manager
  • Tail: exp/{id}/results/runs.log
  • exp-manager diagnoses failures, retries
Phase 3
Analyze
  • Run /analyze-experiment
  • analyze.py → summary.md generated
  • domain-expert (Opus) interprets results
  • slides-maker generates HTML slides
Phase 4
Archive
  • Commit findings to main
  • Run /update-project-skill
  • CHANGELOG entry for all artifacts
  • Pipeline state → back to "dev"

09 / 10
Core Innovation

Project Skill Auto-Generation

Opus Subagent Scans
  • All exp/*/README.md files
  • All docs/papers/ domain knowledge
  • All prompts/ CHANGELOGs
  • Recent git log --oneline -20
  • Previous SKILL.md (preserve custom)
SKILL.md Standard Sections
  • Project Overview & Current State
  • Architecture (code structure)
  • Experiment History Table
  • Key Pitfalls & Lessons Learned
  • Active Prompt Versions
  • Quick Reference
Key Invariants
Pitfalls are append-only — never removed. "Done" experiments never downgraded. Diff shown before write — user approves. CHANGELOG auto-maintained with every update.

Quickstart

Getting Started

  1. 01
    Use as GitHub template
    Click "Use this template" → new repo with all CC infra pre-configured
  2. 02
    Run bootstrap.sh — 4 questions
    Project name · One-line description · Research domain · Compute environment. Idempotent — safe to re-run.
  3. 03
    Open Claude Code, run /new-experiment
    Scaffolds your first exp01a/ directory with full structure. Pipeline state advances to "experiment".
  4. 04
    Run, analyze, iterate
    launch_exp.py → /analyze-experiment → /update-project-skill → commit → repeat
